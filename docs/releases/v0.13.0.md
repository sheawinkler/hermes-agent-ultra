# Hermes Agent Ultra v0.13.0

Release date: 2026-05-23

This release syncs against upstream `NousResearch/hermes-agent` at `7f1b2b4569532d63a7f50e172963da0d4f3082f7` and ports the applicable Rust-side safety and gateway behavior deltas.

## Highlights

- Memory tool writes now refuse external drift in `MEMORY.md` / `USER.md`, preserve the on-disk file, and save a `.bak.*` recovery snapshot before aborting.
- Memory entries are scanned for prompt-injection and secret-exfiltration payloads before persistence.
- Terminal commands that require confirmation now fail closed when no explicit user approval is supplied. Silence is not consent.
- Skill install policy helpers now pin the upstream contract that `--force` cannot override dangerous verdicts for external community/trusted skills.
- Webhook auth rejects `INSECURE_NO_AUTH` on the current public-bind Rust adapter so dynamic or local config cannot accidentally expose unauthenticated routes.
- Gateway status callbacks use a keyed status-update API; Telegram overrides it to edit existing status bubbles instead of appending repeated updates.
- Added upstream-compatible `hermes portal [status|setup]` and `hermes setup --portal` aliases for Nous Portal OAuth setup/status.

## Verification

cargo fmt --all --check
