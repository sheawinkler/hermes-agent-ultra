# Hermes Agent Ultra v0.21.2

Release date: 2026-07-03

`v0.21.2` is a corrective release for published binary confidence. The source
at `v0.21.1` was healthy, but the release archives packaged the legacy `hermes`
wrapper as the payload that the installer renames to `hermes-agent-ultra`. That
can recurse after install instead of launching the canonical CLI.

## What Changed

| Area | Change |
| --- | --- |
| Release packaging | GitHub release builds now compile `--bin hermes-agent-ultra` and package that canonical binary as archive member `hermes`, preserving installer/Homebrew compatibility without shipping the wrapper as the payload. |
| Release smoke | Added `scripts/smoke-release-artifact.sh` to install from published GitHub release assets into a temp directory, preserve any existing upstream `hermes`, reject source-build fallback, and probe core runtime surfaces. |
| Probe safety | Release smoke probes now have bounded per-command timeouts with captured-output diagnostics, so a bad artifact fails deterministically instead of hanging CI. |
| Public readiness | Added a public release-readiness summary generator and Parity Audit integration for zero-pending parity, coverage, and SOTA harness gates. |
| Demo path | Added `docs/demo.md` with the shortest public proof path for release artifacts and source-tree confidence checks. |

## Verification Targets

```bash
python3 scripts/generate-release-readiness-summary.py --repo-root . --check
bash scripts/smoke-release-artifact.sh --version v0.21.2
CARGO_TARGET_DIR=${CARGO_TARGET_DIR:-target} cargo test -p hermes-cli --test e2e_sota_workflow_replay -- --nocapture
CARGO_TARGET_DIR=${CARGO_TARGET_DIR:-target} cargo test -p hermes-source-parity-tests --test global_parity_governance -- --nocapture
CARGO_TARGET_DIR=${CARGO_TARGET_DIR:-target} cargo test -p hermes-source-parity-tests --test sota_harness_contracts -- --nocapture
```

The release workflow also runs the published-artifact smoke after assets are
signed and uploaded for future tags.
