# Hermes Agent Ultra v0.1.0

Release date: 2026-04-28

## Scope
This is a medium-to-major milestone release that promotes Hermes Agent Ultra from the early `0.0.x` line to `0.1.0`, with production hardening and release-channel alignment.

## Highlights
- Secrets hardening pass across CLI, gateway adapters, and security filters.
- Plaintext secret output now requires explicit opt-in via `HERMES_ALLOW_SECRET_STDOUT=1`.
- Gateway startup logs no longer expose token prefixes; they use non-reversible descriptors.
- Expanded Telegram-style token detection in redaction and credential guards.
- Sanitized security fixtures to avoid real-world token namespace overlap.

## Release integrity updates
- Workspace version bumped from `0.0.4` to `0.1.0`.
- Default update-check repository now points to `sheawinkler/hermes-agent-ultra`.
- Workspace repository metadata aligned to this repo.
- Prometheus metrics content-type version now tracks package version dynamically.
- Ancillary release metadata paths (Nix/Homebrew/package headers) aligned to Hermes Agent Ultra branding.
